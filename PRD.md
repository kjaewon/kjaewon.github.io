# 은퇴 카운트다운 및 준비 계획 웹앱 — 제품 요구사항 문서(PRD)

## 1. 개요

### 1.1 목적

- 사용자의 **현재 나이**와 **목표 은퇴 시점**을 기준으로 **은퇴까지 남은 년·월·일**을 계산하여 보여 준다.
- 남은 기간 중 **100일, 200일, 1년, 3년** 등 **특정 기념일(마일스톤)**을 표시한다.
- 은퇴 기간 동안 **준비할 계획(할 일)**을 입력하고, **달성 여부**를 체크할 수 있게 한다.

### 1.2 대상 사용자

- 은퇴 예정자(예: 5~20년 내 은퇴 목표)
- 은퇴까지 남은 기간을 한눈에 보고, 준비 항목을 관리하고 싶은 사용자

### 1.3 제품 범위(In Scope)

- 남은 기간 계산(년/월/일)
- 기념일 목록 표시(100일, 200일, 1년, 2년, 3년, 5년, 10년 등)
- 은퇴 준비 계획 CRUD(생성·조회·수정·삭제)
- 계획별 달성 여부 체크(완료/미완료)
- 단일 사용자 기준 동작(브라우저 로컬 저장 또는 간단 백엔드)

### 1.4 제품 범위(Out of Scope) — 검토 후 조정 가능

- 다중 사용자·회원가입·로그인(필요 시 후속 버전)
- 모바일 전용 네이티브 앱
- 공식 금융/연금 상품 연동

## 2. 사용자 시나리오

| 시나리오     | 설명                                                        |
| ------------ | ----------------------------------------------------------- |
| 남은 기간 확인 | 생년월일(또는 현재 나이)과 은퇴 예정일을 입력하면 "D-XXX일", "X년 X개월 X일" 형태로 표시 |
| 기념일 확인   | 100일, 200일, 1년, 3년 등 남은 기간 기념일 목록과 해당 날짜를 한 화면에서 확인       |

| 계획 수정·삭제 | 잘못 입력한 계획 수정, 불필요한 계획 삭제                                  |

## 3. 기능 요구사항

### 3.1 은퇴까지 남은 기간 계산

- **입력**
  - **은퇴 예정일**: 날짜 선택(년/월/일) — 필수.
  - **현재 날짜 기준**: 서버/브라우저 현재 일자 사용. (선택: 생년월일 입력 시 "현재 나이" 자동 표시)
- **출력**
  - 남은 **일수** (D-XXX).
  - 남은 **년·월·일** (예: 5년 3개월 12일).
  - (선택) 남은 **개월 수**, **주 수** 등 추가 단위.
- **규칙**
  - 은퇴 예정일이 과거이면 "이미 은퇴일이 지났습니다" 등 안내 메시지.
  - 오늘과 같은 날이면 "오늘 은퇴일입니다" 등 축하 메시지.

### 3.2 기념일(마일스톤) 표시

- **정의**: "은퇴일로부터 N일/N년 전"인 날짜를 기념일로 간주.
- **기본 표시 항목(설정 가능하게 할지 여부는 구현 시 결정)**
  - 100일 전, 200일 전, 300일 전
  - 1년 전, 2년 전, 3년 전, 5년 전, 10년 전
- **표시 정보**
  - 기념일 이름(예: "은퇴 100일 전").
  - 해당 **날짜**(년/월/일).
  - (선택) 그 날짜까지 남은 일수(오늘 기준).
- **규칙**
  - 이미 지난 기념일은 "지남" 등으로 표시하거나 목록에서 구분.
  - 미래 기념일만 강조하거나, 전체를 리스트로 나열하도록 선택 가능.

### 3.3 은퇴 준비 계획 관리

- **계획(할 일)**
  - **필수 필드**: 제목(텍스트).
  - **선택 필드**: 설명(메모), 목표 완료일, 우선순위(높음/보통/낮음), 카테고리(예: 재무, 건강, 취미).
- **동작**
  - 계획 **추가**: 제목 입력 후 저장.
  - 계획 **목록 조회**: 추가된 순 또는 완료 여부·날짜 등으로 정렬.
  - 계획 **수정**: 제목·설명·목표일·우선순위 등 편집.
  - 계획 **삭제**: 단일 삭제 + (선택) 완료 항목 일괄 삭제.
- **달성 여부**
  - 각 계획에 **완료 체크박스**.
  - 체크 시 "완료" 처리, 해제 시 "미완료".
  - (선택) 완료일 자동 기록.

### 3.4 데이터 저장

- **1차 목표**: 브라우저 **로컬 스토리지(LocalStorage)** 에 저장하여 별도 서버 없이 동작.
- **확장**: 추후 백엔드(예: REST API + DB) 연동 시 계정당 데이터 이전 가능하도록 구조 고려.

## 4. 비기능 요구사항

- **플랫폼**: 반응형 웹(PC·태블릿·모바일 브라우저).
- **성능**: 초기 로딩 3초 이내, 입력·체크 반응 즉시 반영.
- **접근성**: 키보드 조작 가능, 레이블·대비 등 기본 a11y 고려.
- **브라우저**: 최신 Chrome, Firefox, Safari, Edge.

---

## 5. UI/UX 요구사항

- **메인 화면**
  - 상단: 은퇴 예정일 입력/표시, **남은 기간 대시보드**(D-XXX, X년 X개월 X일).
  - 중단: **기념일 목록**(카드 또는 리스트).
  - 하단(또는 별도 섹션): **은퇴 준비 계획** 목록 + "계획 추가" 버튼.
- **계획 카드/행**
  - 제목, 완료 체크박스, (선택) 수정/삭제 버튼.
  - 완료된 항목은 취소선 또는 흐리게 표시.
- **일관성**: 단일 페이지 앱(SPA) 또는 2~3개 화면(설정·계획 목록·기념일)으로 구성. 네비게이션 명확히.

## 6. 데이터 모델(개념)

- **사용자 설정**
  - `retirementDate`: Date (은퇴 예정일)
  - (선택) `birthDate`: Date (생년월일)
- **기념일**
  - 계산값: 기념일 이름, 기준일(은퇴일 - N일/년), 해당 날짜.
- **계획**
  - `id`, `title`, `completed`(boolean), `createdAt`, (선택) `description`, `dueDate`, `priority`, `category`, `completedAt`.

## 7. 성공 지표(선택)

- 남은 기간이 정확히 계산되는지 검증.
- 기념일 날짜가 은퇴일 기준으로 올바른지 검증.
- 계획 10개 이상 입력 시에도 목록·체크 반응이 빠른지 확인.
